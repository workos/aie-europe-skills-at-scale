# Rehearsal Checklist — Skills at Scale

## Timing checkpoints
- **0:00** Title
- **3:00** `Which would you trust?`
- **7:00** `Get set up`
- **12:00** `Build the Foundation`
- **32:00** `Make It Smarter`
- **57:00** `Skills Beyond the Editor`
- **72:00** `Close`
- **80:00** End

If you're off by **>2–3 min**, trim live narration.

---

## Block 1 — Problem + What Skills Are
**Goal:** pain → proof → framing, fast

Checklist:
- [ ] Show-of-hands lands quickly
- [ ] `Who we are` stays under 30 sec
- [ ] `The problem` under 1 min
- [ ] Hit `Which would you trust?` by ~3:00
- [ ] Don’t over-explain `Anatomy of a skill`
- [ ] Reach setup by ~7:00

Trim first if long:
- [ ] extra credibility details
- [ ] deep anatomy explanation

Never trim:
- [ ] `Which would you trust?`

---

## Block 2 — Setup
**Goal:** get people running with minimal stage talk

Checklist:
- [ ] Verbal instructions under 90 sec
- [ ] One presenter talks, one roams
- [ ] QR is readable from room
- [ ] `setup.sh` works cleanly
- [ ] “Health score? You’re ready.” is clear
- [ ] People know what tool/path they’re using

Fallback:
- [ ] if setup drags, move helpers into room immediately

---

## Block 3 — Build the Foundation
**Goal:** teach just enough, then get people editing

Checklist:
- [ ] Constraints explanation stays practical
- [ ] `!` backtick slide lands clearly
- [ ] Description/routing test is crisp
- [ ] `Your turn` gives exact next steps
- [ ] Attendees know:
  - [ ] what file to open
  - [ ] what to change
  - [ ] what to run
- [ ] Build time is protected

Trim first if long:
- [ ] philosophy of constraints

Protect:
- [ ] scripts explanation
- [ ] hands-on time

---

## Block 4 — Make It Smarter
**Goal:** explain phases + confidence without getting abstract

Checklist:
- [ ] `Progressive disclosure` under control
- [ ] Explain clearly: in-file phases = structure, external files = real gating
- [ ] Confidence scoring feels useful, not ceremonial
- [ ] Ideation example is delivered in 1 tight before/after
- [ ] `Iteration loop` is a reset, not a lecture
- [ ] Reach hands-on quickly

Trim first if long:
- [ ] extra phase-gating theory
- [ ] anecdote length

Protect:
- [ ] build/re-run/compare time

---

## Block 5 — Skills Beyond the Editor
**Goal:** keep energy high after long hands-on blocks

Checklist:
- [ ] `Level up your scripts` instruction is simple: pick one, paste, run
- [ ] People don’t stall reading all options
- [ ] `Same skill, everywhere` lands as one thesis:
  - [ ] same markdown
  - [ ] different runtimes
- [ ] `Measurement matters` stays sharp and short
- [ ] Block doesn’t turn into another lecture

Trim first if long:
- [ ] extra portability examples
- [ ] measurement setup

Protect:
- [ ] one fun script exercise
- [ ] one crisp measurement takeaway

---

## Block 6 — Close
**Goal:** payoff → practical next step → callback

Checklist:
- [ ] `share.sh` works
- [ ] projector flow is ready
- [ ] pair-share works even if projector fails
- [ ] install/reuse slide stays brief
- [ ] final callback doesn’t feel rushed
- [ ] end on time

Fallback if sharing breaks:
- [ ] pair-share only
- [ ] ask 2 volunteers to read a favorite constraint/script aloud

---

## Components check
Before rehearsal, verify:
- [ ] `BadGoodCompare` animation timing feels fast enough
- [ ] `SkillAnatomy` is readable and not over-detailed
- [ ] `FolderTree` fits and explains quickly
- [ ] `ScriptMenu` invites action, not browsing
- [ ] `PortabilityGrid` is understandable at a glance

Rule:
- [ ] no component should need >30 sec of explanation

---

## Speaker choreography
Checklist:
- [ ] one primary presenter per block
- [ ] handoffs happen at section boundaries
- [ ] during setup/hands-on: one on mic, one roaming
- [ ] “While you work” riffs are time-boxed
- [ ] no critical instructions are hidden inside riffs

---

## Pre-show logistics
- [ ] `share.sh` tested end-to-end
- [ ] `.share-config` confirmed
- [ ] QR works
- [ ] repo clone/setup works on a clean machine
- [ ] demo repo chosen
- [ ] fallback repo chosen
- [ ] fonts/assets/components render correctly
- [ ] projector resolution checked
- [ ] speaker notes visible as expected

---

## 3 rehearsal passes
### Pass 1 — Timing
- [ ] hit checkpoints
- [ ] note overruns
- [ ] don’t polish wording

### Pass 2 — Choreography + tech
- [ ] practice handoffs
- [ ] test components
- [ ] test setup
- [ ] test share flow

### Pass 3 — Full run
- [ ] full energy
- [ ] no stopping
- [ ] use fallback decisions live

---

## Emergency trims
If running behind:

**Trim first**
- [ ] extra credibility detail
- [ ] deep anatomy explanation
- [ ] long anecdotes in While-you-work
- [ ] extra portability examples
- [ ] measurement setup

**Do not trim**
- [ ] bad vs good contrast
- [ ] scripts concept
- [ ] hands-on edit/run cycles
- [ ] closing callback
